ABOUT THE DATASET:
This dataset contains structured system log records captured from a multi-team e-commerce platform. Each log entry represents a real-time incident or issue detected across various services and infrastructure layers. Logs are generated by developers, monitoring agents, and automated alerting systems throughout the working day.

FEATURES:
log_id, timestamp, user_id, team, module_accessed, action, issue_detected, issue_type, error_code, error_message, ticket_id, priority, assigned_team, status, response_time_ms, cpu_usage_pct, memory_usage_mb

PROBLEM STATEMENT:
When an incident is logged, it is not always immediately clear which department should own and resolve it. Misrouted tickets waste engineering time and delay resolution. This project performs Exploratory Data Analysis (EDA) on the log data to understand patterns and distributions across teams, extracts discriminative keywords per department using TF-IDF, detects and treats outliers in system metrics, and trains a multi-class classification model to automatically predict the correct assigned department (Auth, Backend, Database, DevOps, Frontend, Payment, Search, Security) from raw log attributes, enabling faster and accurate ticket routing with no manual triage.
